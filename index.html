02 — Process

How Ambisonics
actually works

Ambisonics samples the soundfield using microphone patterns that correspond to the real-valued spherical harmonics. Instead of routing audio to fixed speakers, you describe the soundfield mathematically — then project it to whatever speakers you have.

01
Capture
Four capsules in a tetrahedral arrangement (A-format) capture the full soundfield. Combined into B-format: W (omni) + X, Y, Z (figure-8 at right angles).
02
Encode
A mono source at azimuth θ, elevation φ is encoded: W = S·0.707, X = S·cos(θ)·cos(φ), Y = S·sin(θ)·cos(φ), Z = S·sin(φ).
03
Transport
The B-format signal (or higher-order equivalent) is stored or streamed. Speaker layout is irrelevant at this stage — pure soundfield data.
04
Decode
A decoder projects the spherical harmonic components onto any speaker arrangement. New layout? New decoder. Same master recording always.
First-order ambisonics — encoding equations
W = S × 0.707                             // omnidirectional (pressure)
X = S × cos(azimuth) × cos(elevation)   // front-back axis
Y = S × sin(azimuth) × cos(elevation)   // left-right axis
Z = S × sin(elevation)                   // up-down axis
Decoding to 4-speaker quad layout
LF = W + X + Y        // Left Front
RF = W + X − Y        // Right Front
LB = W − X + Y        // Left Back
RB = W − X − Y        // Right Back

04 — Resolution

Higher orders,
higher resolution

Each increase in order adds more spherical harmonic components, improving the angular resolution of the soundfield representation. Higher orders require more channels but give a larger "sweet spot" and sharper imaging.

Channel count formula
Channels = (order + 1)²

Order 0: (0+1)² =  1  channel   ← mono
Order 1: (1+1)² =  4  channels  ← B-format (W, X, Y, Z)
Order 2: (2+1)² =  9  channels
Order 3: (3+1)² = 16  channels
Order 4: (4+1)² = 25  channels
Order 5: (5+1)² = 36  channels
Order 6: (6+1)² = 49  channels
Order 7: (7+1)² = 64  channels  ← practical HOA limit
